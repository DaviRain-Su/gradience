# GRA-138: [Arch] 白皮书 §8.4 MagicBlock 重定位为执行层

## Description
将白皮书 §8.4 中 MagicBlock Ephemeral Rollups 从"通信协议"重定位为"Solana 结算层的可选增强组件"。ER 用于高频场景弹性扩展，PER 用于 sealed 模式隐私执行，VRF 用于 Judge 随机选择。MagicBlock 不是独立的"执行层"，因为 Agent 执行任务是 off-chain 的，MagicBlock 解决的是链上交易的效率和隐私问题。

## 具体变更
- "Option A: Ephemeral Rollups (Current Primary)" 重写为 "Execution Layer: MagicBlock"
- 明确三个子组件: ER (任务执行), PER (sealed 模式/TEE), VRF (Judge 随机)
- 从 Transport Adapters 表格中移除 MagicBlock (它不是 transport)
- 更新 "Option C: Hybrid" 映射表，把 ER 放在执行层而非通信层
