# Phase 1: Product Requirements Document (PRD)

## AI Native 租房平台

> **版本**: v1.0  
> **日期**: 2026-04-09  
> **状态**: 定稿  
> **作者**: Product Team

---

## 1. 项目概述

### 1.1 一句话定位

**AI Native 租房平台**是一个面向公寓运营商和中介的 AI 房源发布与管理工具，同时为租客提供浏览和找房入口。平台以 Coze（扣子）Agent 为对话前台和渠道分发器，自研系统承载核心业务逻辑、数据资产和支付结算。

### 1.2 核心理念

- **Processize before Productize**：先让中介把平台当工具用起来，再逐步产品化、平台化
- **AI Native**：用自然语言替代复杂表单，让"对话即操作"
- **渠道优先**：借助 Coze/字节生态，零门槛覆盖微信公众号、抖音私信等核心流量渠道

### 1.3 核心约束

1. 排除区块链方案
2. 支付走国内支付宝（不走 Stripe/X402）
3. MVP 阶段不做信用评价体系
4. Coze 只做"语言理解和渠道分发"，业务事实全部落入自研系统

## 2. 目标与愿景

### 2.1 短期目标（MVP，0-3 个月）

- 让 5-10 个种子中介/公寓运营商能够**通过微信对话免费录入房源、生成分享链接、追踪线索**
- 验证"AI 导入 + 分享归因 + 数据反馈"这一供给侧工具链路是否成立
- 跑通从房源录入到租客浏览的完整闭环

### 2.2 中期目标（3-6 个月）

- 供给侧用户扩展至 50+，推出付费订阅版
- 上线微信小程序，承接抖音/微信双渠道流量
- 建立中介邀请码裂变机制，实现自然增长

### 2.3 长期愿景（6-12 个月）

- 成为中小型公寓运营商和独立中介的**首选 AI 房源管理工具**
- 用户密度足够后，向"租房生活广场"演进（内容社区 + 区域生活指南）
- 探索 AI 租房合同生成等增值服务

## 3. 用户画像

### 3.1 供给侧：公寓运营商 / 独立中介

**统一角色定义**：
- 公寓运营商（管理整栋/整层公寓）
- 代理公寓的独立中介（兼职或全职）
- **边界模糊，平台统一对待，都是付费用户**

**核心痛点**：
1. 推广成本高：挂贝壳抽佣高，发朋友圈不够专业
2. 管理工具差：自建小程序成本极高，Excel 管理混乱
3. 效果不可追踪：不知道发出去的内容有多少人看、有没有人联系

**核心需求**：
- 快速录入房源（最好是发图片/说话就能录）
- 生成专业的分享链接/海报（替代朋友圈截图）
- 看到清晰的数据反馈（点击量、联系量）
- 成本比贝壳低，效率比 Excel 高

### 3.2 需求侧：租客

**核心痛点**：
1. 公寓信息分散（小红书、微信群、贝壳来回切换）
2. 找房效率低：很多房源已租出但还没下架
3. 怕被虚假房源骗

**核心需求**：
- 快速浏览真实房源
- 直接联系到发布房源的中介（不想被平台转卖线索）
- 能够明确表达需求（区域、预算、户型、特殊要求）

**角色属性**：
- **永久免费**，是平台的流量来源和裂变介质
- 通过中介的分享链接进入平台，或关注公众号后主动找房

## 5. 核心功能需求

### 5.1 供给侧：ListingAgent（微信公众号 Bot）

#### FR-S1: 注册与绑定
- 中介关注公众号后，发送手机号完成注册
- 首次录入房源前，Bot 推送《房源信息真实性承诺》，需点击"同意"后方可继续
- 系统将中介手机号与微信 OpenID 绑定，生成唯一的 `agent_id`

#### FR-S2: 房源录入（AI 导入 + 人工确认）
- 中介可通过文字、图片、Excel 文件发送房源信息
- Coze Workflow 调用 OCR/LLM 提取结构化字段：
  - 必填：地址/小区名、户型、面积、租金、联系人/电话
  - 选填：楼层、朝向、配套设施、可入住时间、宠物政策、房源图片
- 提取结果以结构化卡片形式返回，附带【确认发布】和【修改信息】按钮
- 点击【确认发布】后，房源状态变为"已上架"
- 点击【修改信息】后，Bot 逐字段询问并更新
- 未确认的房源可在自研后台批量编辑

#### FR-S3: 分享链接生成
- 中介可通过自然语言（"生成 SOHO 现代城的链接"）或点击菜单触发
- 系统生成短链，格式示例：`apt.io/s/{short_code}?agent={agent_id}&listing={listing_id}`
- 同时生成带二维码的海报图片，方便发朋友圈
- 每个链接永久有效，直到房源下架

#### FR-S4: 线索查询
- 中介可随时在对话中查询数据："这周数据怎么样？"
- Bot 返回关键指标：
  - 总点击量
  - 有效点击量（停留 > 10 秒，且通过 IP + 设备指纹去重）
  - 联系按钮点击数
  - 各房源的热度排名
- 数据时间范围支持"今天/本周/本月"

#### FR-S5: 房源管理（对话 + 后台双入口）
- 对话内支持："下架 502"、"把 SOHO 现代城租金改成 8000"
- 后台支持：房源列表、编辑、批量上下架、待确认房源处理

### 5.2 需求侧：RenterAgent（微信公众号 Bot + H5）

#### FR-R1: 被动浏览（主力入口）
- 租客通过中介分享的短链/海报进入 H5
- H5 展示房源详情：图片轮播、基础信息、配套设施、位置描述
- 页面底部固定展示该中介的微信二维码和电话
- 支持"一键拨号"和"长按识别二维码"
- H5 页面需做 SSG/ISR，URL 对搜索引擎友好

#### FR-R2: 主动找房（公众号 Bot 辅助）
- 租客关注公众号后，可用自然语言描述需求
- Bot 提取结构化条件：区域、预算区间、户型、入住时间、特殊要求（如养猫）
- 调用自研搜索接口，返回 2-5 套匹配房源
- Bot 以自然语言形式回复推荐结果，并支持"回复编号查看详情"或"预约看房"

#### FR-R3: 注册登录
- 租客首次使用 Bot 或点击"收藏/预约"时，引导手机号注册
- 注册后获得 `user_id`，用于后续的线索归因和对话上下文

### 5.3 平台侧：自研系统

#### FR-P1: Plugin 网关
- 提供标准化 REST API 供 Coze Workflow/Bot 调用
- 所有接口需校验 Coze 请求签名，防止伪造
- 接口列表：
  - `POST /plugin/listings/create` — 创建房源（待确认/已上架）
  - `POST /plugin/listings/update` — 更新房源
  - `POST /plugin/listings/search` — 搜索房源
  - `POST /plugin/listings/delete` — 下架房源
  - `POST /plugin/share/generate` — 生成短链
  - `POST /plugin/leads/track` — 记录线索事件
  - `POST /plugin/leads/report` — 查询线索报表
  - `POST /plugin/users/profile` — 查询/更新用户档案

#### FR-P2: 短链与归因服务
- 短链生成：基于 Redis 的短码映射
- 点击追踪：记录 UV（去重）、PV、停留时长、联系点击
- 防刷机制：
  - 同一 IP + 设备指纹在 1 小时内多次点击只算 1 次 UV
  - 停留时间 < 3 秒的点击标记为"无效"，不进入有效指标
- 联系人展示：H5 页面根据短链中的 `agent_id` 动态渲染中介信息

#### FR-P3: 支付系统
- 仅对供给侧收费，采用包月订阅制
- 支付方式：支付宝手机网站支付
- 订阅状态管理：生效中、已过期、待续费
- 过期后自动降级为免费版（保留已上架房源但限制新增）

#### FR-P4: 合规机制
- 房源发布前强制展示《房源信息真实性承诺》
- H5 详情页底部设置"举报虚假房源"入口
- 平台保留 24 小时内下架违规房源的权利
- 所有页面底部展示法律免责声明

## 6. 产品流程

### 6.1 供给侧核心流程

```
[中介关注公众号]
    ↓
[发送手机号注册]
    ↓
[签署真实性承诺]
    ↓
[与 ListingAgent 对话]
    ↓
[发送房源信息 ← 文字/图片/Excel]
    ↓
[Coze Workflow 提取结构化信息]
    ↓
[Bot 返回确认卡片]
    ↓
    ├─ 【确认发布】→ 房源上架
    └─ 【修改信息】→ 逐字段编辑
    ↓
[生成分享短链/海报]
    ↓
[中介分享到微信群/朋友圈/小红书]
    ↓
[租客点击链接浏览]
    ↓
[中介在 Bot/后台查看线索数据]
    ↓
[免费额度用完 → 引导支付宝订阅]
```

### 6.2 需求侧核心流程

```
[租客在朋友圈看到中介分享的海报]
    ↓
[扫码或点击短链进入 H5]
    ↓
[浏览房源详情]
    ↓
    ├─ 感兴趣 → 长按二维码加中介微信 / 一键拨号
    └─ 想继续看 → 关注公众号
    ↓
[在公众号与 RenterAgent 对话]
    ↓
[描述租房需求]
    ↓
[Bot 返回匹配房源推荐]
    ↓
[回复编号查看详情/预约看房]
```

## 7. 商业模式与定价

### 7.1 付费方与免费方

- **付费方**：供给侧（公寓运营商 + 独立中介）
- **免费方**：租客（永久免费，保证流量端不设置门槛）

### 7.2 冷启动策略

**第一阶段（0-6 个月）：全部免费**
- 供给侧无门槛使用全部基础功能
- 目标是让中介养成使用习惯，积累房源和线索数据
- 用实际数据（"您的链接带来 23 次点击"）证明平台价值

### 7.3 付费订阅模式

| 功能 | 免费版 | 付费版（包月） |
|------|--------|---------------|
| 录入房源数量 | 5 套 | 不限 |
| 生成分享链接 | 3 条/月 | 不限 |
| 数据报表 | 仅总点击量 | 完整线索漏斗、有效点击、转化追踪 |
| 房源展示排序 | 普通 | 优先展示 |
| 价格 | ¥0 | 待定（建议 ¥29-99/月，根据种子用户访谈调整） |

### 7.4 邀请码裂变激励

- 每位注册中介获得专属邀请码
- 邀请一位新中介注册并发布至少 1 套房源：
  - 邀请人获得：+3 套免费房源额度 或 +5 条当月分享链接
  - 被邀请人获得：同上

## 9. 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Coze 平台稳定性/政策变动 | 高 | 核心业务逻辑全部在自研 Plugin 网关和业务中台，Coze 仅做渠道层，切换成本可控 |
| AI 房源提取准确率不足 | 中 | 强制加入"人工确认"环节，后台支持批量编辑，不追求 100% 自动化 |
| 中介冷启动意愿低 | 高 | 完全免费 + 种子用户深度陪跑 + 邀请码裂变激励 |
| 虚假房源法律风险 | 中 | 真实性承诺书 + 举报下架机制 + 法律免责声明 + 不介入资金托管 |
| 租客流量不足 | 中 | 依赖中介的社交网络自然传播，平台本身不投流，而是给中介提供"更好的发朋友圈工具" |
| 竞品（贝壳/58）快速跟进 | 低 | 错位竞争：不做大平台流量分发，而是做中小中介的"AI 效率工具" |

## 10. 后续路线图

### Phase 1：MVP 验证（0-3 个月）
- Coze 接入微信公众号
- 自研 H5 + 中介后台上线
- 房源导入（带确认）、分享链接、线索追踪跑通
- 支付宝订阅支付上线
- 从 1-2 个种子中介开始冷启动

### Phase 2：增长与扩展（3-6 个月）
- H5 核心页移植为微信小程序
- 开通抖音私信 Bot，配合短视频内容引流
- 上线中介邀请码裂变机制
- 推出付费订阅版
- 扩大供给侧至 50+ 中介/公寓

### Phase 3：平台化（6-12 个月）
- 上线小区房源独立分类
- 引入租客评价和社区内容
- 探索 AI 租房合同生成等增值服务
- 接入微信客服、飞书等更多渠道

## 11. 关键决策记录

| 决策项 | 结论 | 理由 |
|--------|------|------|
| 是否使用 Coze | **是** | 核心优势是渠道准入（微信公众号 + 抖音私信绿色通道），而非技术能力替代 |
| Coze 与自研边界 | Coze = 语言+渠道；自研 = 事实+支付 | 避免业务逻辑失控，确保核心资产自主可控 |
| 统一 Agent vs 独立 Agent | **统一 Agent** | MVP 最高效，通过 `user_id` 区分用户身份 |
| 先做 H5 还是小程序 | **先 H5** | 传播路径最短，小程序在 Phase 2 移植 |
| 租客端搜索 vs 对话 | **搜索为主，对话为辅** | 租房是强目的性行为，降低用户认知门槛 |
| 付费时点 | **先用后付** | 降低冷启动门槛，用效果数据驱动转化 |
| 房源导入准确性 | **AI 提取 + 人工确认** | 保证用户体验，避免错误信息直接上架 |
| 技术网关框架 | **Fastify** | 性能优于 Express，适合短链高并发场景 |
