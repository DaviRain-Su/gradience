## 1.2 问题定义（必填）

### 要解决的问题

多 Agent 协作缺少统一的消息信封、可结算微支付、以及可审计的任务分解/分配协议，导致无法安全规模化协同。

### 当前状态

- AgentM 只有前端内存/BroadcastChannel 级别的 A2A 邀请 stub。
- 没有链上消息可验证层（签名信封 + 发现广播）。
- 没有链上可争议的微支付通道。
- 没有标准化的子任务竞标/分配协议与 Agent Arena/Chain Hub 联动。

### 目标状态

构建可落地的三层 A2A 协议：

1. **A2A-0 消息层**：链上消息索引 + 签名信封 + Agent 发现广播。
2. **A2A-1 微支付通道**：链上 open/close/dispute，链下签名状态更新。
3. **A2A-2 任务分解**：大任务拆分、A2A 广播、竞标、分配、交付与结算闭环。

## 1.4 功能范围（必填）

### 做什么（In Scope）

- [x] 新建 A2A Channel Program（Pinocchio/no_std），支持消息、通道、子任务三类状态账户
- [x] A2A-0：消息 Thread + MessageEnvelope 上链索引，签名信封校验，Agent 发现广播
- [x] A2A-1：支付通道 open/close/dispute/resolve，链下状态签名 + 链上最终结算
- [x] A2A-2：子任务订单、竞标、分配、交付、结算状态机
- [x] TypeScript A2A Runtime（relay + SDK）对接现有 Agent Arena/Chain Hub

### 不做什么（Out of Scope）

- 不做跨链桥资产转移（仅 Solana 域内结算）
- 不做端到端加密密钥托管系统（仅定义 envelope 签名与哈希约束）
- 不做复杂拍卖机制（v0.1 使用固定规则竞标）
- 不做去中心化消息中继网络（v0.1 允许中心化 relay）
