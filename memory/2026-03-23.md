## X/Twitter RSS Feed 调研
- 用户想要将任意 X 账户转为 RSS 订阅
- 尝试了多种方法：xcancel.com、Nitter、RSS.app、XRSS
- 结论：X 反爬严格，目前可行方案：
  1. **XRSS** (自托管，需 X 账号) - 最稳定
  2. **RSS.app** (付费托管) - 最简单
  3. **Inoreader** (免费 tier 150 feeds) - 原生支持 X 搜索
  4. **xcancel.com** 半失效，需要特殊 User-Agent

## Agent Matrix 一人公司架构
- 设计了多 Agent 协作架构：CEO、Engineer、PM、Creator、Research、Auditor、Support、Deploy
- 核心原则：Agent 间通过标准协议通信，人类只在关键节点介入
- 角色分层：决策层、执行层、支持层、接口层
- 协作模式：链式(Chain)、并行(Parallel)、迭代(Iterative)

## 基础设施发现
### db9.ai
- 面向 Agent 的存储平台：文件系统 + PostgreSQL 混合
- 支持百亿级免费 Agent 租户，一虾一库
- Agent 可以用文件接口(cp)或 SQL 接口操作同一数据
- 每个邮箱对应一个 db9 实例的案例 (mails.dev)

### mem9.ai
- OpenClaw 的永续记忆服务，解决 MEMORY.md compact 失忆问题
- 云端持久化，向量+全文检索
- 一虾一库，独立加密，支持迁移
- 安装方式：让 Agent 读取 https://mem9.ai/SKILL.md

## Thorsten Ball "软件正在变成液体"
- 软件从「固体」(版本号、releases) 变成「液体」(即用即走、无版本)
- 案例：$25k 专业扭矩扳手 vs $1k 蓝牙扳手 + AI 脚本
- 未来：连脚本都不存储，直接自然语言指令，代码出现→工作→消失
- Ghostling：600 行 C 代码终端模拟器，作者"一行没写，全是 AI"
- Node.js VFS：14,000 行 PR， holiday side project，无 AI 不可能完成
- 启示：Agent Matrix 应该接受「液体代码」为默认形态
