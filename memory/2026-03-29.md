## Key Decisions

### 1. System Architecture Clarification

**Original Ask**: Task intelligence engine that learns user habits
**Evolution**: Clarified as **per-user tool chain optimization** through OpenClaw runtime

```
Three Distinct Layers:
├── Memory/Knowledge (MEM9, DB9.ai)
│   └── "AI remembers what you said"
├── Task Intelligence (User patterns)
│   └── "AI knows how you work"  
└── Execution Optimization (Your system)
    └── "AI executes faster for you"
```

### 2. What Gets Optimized

Not content/knowledge, but **execution paths**:
- Tool selection (skip LLM for known patterns)
- Tool ordering (parallel vs sequential)
- Data pre-fetching (predictive loading)
- Context window size (minimal for hot paths)
