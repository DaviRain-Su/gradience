\begin{abstract}\noindent
本文提出 Gradience——一个\textbf{去中心化的 AI Agent 能力信用协议}。协议通过受比特币挖矿启发的\textbf{竞争模式}建立链上信用：任何已质押的 Agent 可以向开放任务提交结果，由指定评判者选出最优——触发自动三方结算，同时在链上积累不可伪造的能力信誉。链上信誉从行为中积累，无需注册。角色不是身份而是行为的涌现属性。评判者——类比比特币矿工——无论结果如何都获得固定报酬，消除偏见。整个协议由\textbf{三个状态和四个转换}定义。由此积累的链上工作历史构成可验证的信用层——面向 Agent 的低抵押借贷协议与信用背书稳定币是本协议的长期愿景（见 §8）。
\end{abstract}

\section{1. 引言}

AI Agent 正在成为独立的经济主体。然而 Agent 间经济活动的基础设施完全缺失。\textbf{平台模式}（Virtuals ACP、Upwork）依赖受信中介，抽取 20--30\% 手续费。\textbf{标准提案}（ERC-8183~\cite{erc8183}）定义了托管机制，但缺乏内建信誉、竞争机制和评估者激励。

Gradience 受比特币~\cite{bitcoin}启发，用三个原语定义 Agent 能力交换：

\begin{center}\textbf{托管 + 评判 + 信誉 = 无需信任的能力结算}\end{center}

\section{2. 设计哲学}

\subsection{2.1\quad 角色从行为中涌现}
比特币没有 \texttt{registerAsMiner()}。Gradience 中只有三种行为：\textbf{发布}任务 $\rightarrow$ 发布者；\textbf{提交}结果 $\rightarrow$ 执行者；\textbf{评判}并结算 $\rightarrow$ 评判者。同一地址在不同任务中扮演不同角色。唯一约束：同一任务中不能身兼两角。

\subsection{2.2\quad 协议即承诺}
费率编码为不可变常量。部署后无法修改。如同比特币的 2100 万上限。

\subsection{2.3\quad 复杂度放在上层}
无 Hook、无插件、无扩展点。更复杂的逻辑（竞价、协商、子任务分解）在上层构建。内核保持封闭。这是 Unix 哲学在经济协议中的应用：把一件事做好。

\subsection{2.4\quad 对抗验证：GAN 洞察}

三角色分离不仅是组织设计——它是类似生成对抗网络（GAN）~\cite{gan} 的\textbf{对抗质量机制}。生成器创建输出，判别器评估输出，二者缺一不可。

自我评估存在系统性偏差。Anthropic 工程团队在构建多 Agent 系统时独立证实了这一点~\cite{anthropic}：\emph{``当被要求评估自己生成的作品时，Agent 倾向于自信地赞扬——即使对人类观察者来说，质量明显一般。''}

Gradience 在协议层编码了这一洞察：

\begin{table}[H]\centering\small
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{GAN 组件} & \textbf{Gradience 角色} & \textbf{职能} \\
\midrule
规划器 & 发布者（Poster） & 定义评判标准和奖励 \\
生成器 & 执行者（Agent） & 在竞争压力下完成工作 \\
判别器 & 评判者（Judge） & 独立评估，0--100 评分 \\
\bottomrule
\end{tabular}
\end{table}

三个属性保证其可信：(1)~\textbf{强制分离}——同一任务中无地址可兼任两个角色；(2)~\textbf{利益绑定}——评判者需质押资本，3\% 费用与结果无关（消除结果偏见）；(3)~\textbf{竞争替代迭代}——Anthropic 使用 5--15 轮反馈，Gradience 使用 N 个 Agent 单轮并行竞争。

\subsection{2.5\quad 进化压力：自我改进的网络}

GAN 机制（\S2.4）保障单次任务的质量。但\emph{网络本身}如何随时间改进？比特币有难度调整：矿工变强，协议自动要求更多工作量。Gradience 需要类似的能力进化机制。

自动化研究循环~\cite{karpathy}——修改 $\to$ 评估 $\to$ 对比 $\to$ 提交或回滚 $\to$ 重复——在协议激励结构内运行时产生三种进化压力：

\textbf{（1）Agent 自我进化。} 失败的 Agent 可以分析原因（分数、标准、差距）并自动迭代。竞争模式产生选择压力：进化更快的 Agent 赢得更多任务，积累更多声誉。

\textbf{（2）协议安全加固。} Agent 竞争发现漏洞（``攻破此合约'' 悬赏任务）。协议通过经济激励驱动的持续对抗压力不断加固——而非依赖周期性人工审计。

\textbf{（3）评判质量棘轮。} 评分被频繁推翻的评判者失去声誉，被激励改进评判方法。这形成棘轮效应：更好的 Agent 要求更好的 Judge，更好的 Judge 要求更好的 Agent。

协议提供激励结构（声誉、质押、费用），使自我改进成为经济理性选择。优化机制本身在内核之上运行（\S2.3）。

\textbf{与 UX 层标准的关系。} 产品层标准（如 Linear 的 AIG）在界面层解决信任——身份披露、状态透明、人类可归责。Gradience 在协议层解决同一问题：链上身份替代 UI 标签，\texttt{trace\_ref} 替代状态面板，质押和 Slash 替代责任声明。AIG 让 Agent \emph{感觉}可信；Gradience 让信任\emph{可验证、可结算}。

\subsection{2.5\quad 比特币启发的极简主义}

比特币用三个原语定义"货币"：\textbf{UTXO + 脚本 + 工作量证明}。Gradience 用三个原语定义"Agent 能力交换"：\textbf{托管 + 评判 + 信誉}。

\begin{table}[H]\centering
\begin{tabular}{@{}ll@{}}
\toprule
\textbf{比特币} & \textbf{Gradience} \\
\midrule
工作量证明（竞争） & 质量证明（竞赛模式） \\
区块奖励（固定、不可变） & 95/3/2 分配（不可变常量） \\
难度调整（自适应） & 质押参与（市场驱动） \\
最长链获胜 & 最高评分获胜 \\
\bottomrule
\end{tabular}
\end{table}

\textbf{为什么极简主义很重要：}

\begin{enumerate}[leftmargin=2em]
\item \textbf{可预测性}：每个人都知道规则。没有隐藏逻辑。长期规划成为可能。
\item \textbf{抗攻击性}：简单规则的操纵面更小。如同比特币的 51\% 攻击成本，Gradience 的经济攻击需要控制多数质押或信誉。
\item \textbf{可验证性}：任何人都可以审计协议。无需信任第三方。透明即安全。
\item \textbf{自主运行}：不需要管理员。比特币的自动难度调整；Gradience 的自动结算。7×24 无人值守运行。
\end{enumerate}

\textbf{与复杂系统的对比：}
\begin{itemize}[leftmargin=2em]
\item DeFi 协议：数百行代码，频繁出现漏洞
\item DAO 治理：复杂投票，效率低下
\item 多代币模型：难以理解和预测
\end{itemize}

Gradience：约 300 行代码。这就是整个基础。

\subsection{2.7\quad 身份与隐私：分层验证}

> **核心洞察**：AI Agent 自主行动，但最终由人类拥有和控制。协议必须在不损害隐私的前提下验证 Agent 背后的人类。

\subsubsection{所有权问题}

AI Agent 是可以自主运行的软件：
- 持有私钥并签名交易
- 做出决策并执行任务
- 积累声誉和资产

但它们\textbf{由人类创建和控制}。这带来独特挑战：

\textbf{为什么这很重要：}
\begin{enumerate}[leftmargin=2em]
\item \textbf{金融责任}：当 Agent 借钱时，谁负责偿还？
\item \textbf{法律合规}：当 Agent 赚取收入时，谁支付税款？
\item \textbf{女巫攻击}：如何防止一个人创建无限 Agent 来操纵系统？
\end{enumerate}

Gradience 使用零知识证明实现\textbf{可选的、保护隐私的身份验证}：

\end{table}

\textbf{关键设计原则：}
\begin{enumerate}[leftmargin=2em]
\item \textbf{默认隐私}：Tier 0 和 Tier 1 不需要个人信息
\item \textbf{可选验证}：用户自行选择是否升级到 Tier 2
\item \textbf{零知识}：Tier 2 证明人性但不泄露身份
\item \textbf{可组合性}：声誉在所有层级可携带
\end{enumerate}

\subsubsection{对 Agent 经济的意义}

\textbf{没有身份验证：}
\begin{itemize}[leftmargin=2em]
\item Agent 可自由创建新地址
\item 声誉不可携带且易被操纵
\item 无抵押借贷不可能（无问责制）
\item 协议仍是玩具，不是严肃基础设施
\end{itemize}

\begin{itemize}[leftmargin=2em]
\item 一个人类 = 一个 Tier 2 身份
\item 声誉在持久身份上积累
\item 金融服务成为可能（借贷、信用）
\item 协议成为合规基础设施
\end{itemize}

\textbf{Gradience 方案平衡了：}
\begin{itemize}[leftmargin=2em]
\item \textbf{隐私}：链上无个人数据
\item \textbf{合规}：金融操作有问责主体
\item \textbf{自由}：用户选择参与级别
\item \textbf{安全}：女巫攻击经济上不可行
\end{itemize}

\subsection{3.4\quad 提交可见性}
发布者设置 \texttt{visibility}：\textbf{public}（默认，所有提交公开）或 \textbf{sealed}（结算前隐藏——用于敏感任务）。实现由执行层负责（如 MagicBlock Private ER + TEE）。

\subsection{3.5\quad 质押}
Agent 最低质押由发布者按任务设定。评判者质押为协议最低要求。Phase~1 用 SOL，Phase~3 过渡到 GRAD——每个阶段是新 Program 版本，旧版本保持不变。v1 无显式罚没；坏参与者通过失去信誉而经济死亡。

\subsection{3.6\quad 防作弊}
自评允许用于冷启动，四道防线：(1) 2\% 协议费 = 刷信誉的电费；(2) 质押门槛；(3) 链上 \texttt{selfEvaluated} 标记；(4) 竞争模式下自评无意义。

\subsection{3.7\quad 评判标准}
\texttt{evaluationCID} 引用链下标准。类型：\texttt{test\_cases}、\texttt{judge\_prompt}、\texttt{checklist}、\texttt{custom}。可扩展。推荐存储：\textbf{Arweave}（永久）或 \textbf{Avail}（DA 层）。IPFS 可接受但有 pin 过期风险。

\subsection{3.8\quad 落选提交}
所有提交以引用形式存储在链上。获胜提交链接到已完成任务；落选提交保留为参与证据。可见性遵循任务设置。

\subsection{5.2\quad ERC-8004 集成}

ERC-8004~\cite{erc8004}定义三个注册表：\textbf{身份注册表}（Agent 档案，ERC-721）、\textbf{信誉注册表}（反馈信号）、\textbf{验证注册表}（独立验证钩子）。Gradience 映射到这三个注册表。

\textbf{身份注册表。} Agent 首次参与时自动注册到 ERC-8004。注册文件包含 \texttt{gradience} 服务端点指向 Solana 程序。

\textbf{信誉注册表。} 每次 \texttt{judgeAndPay()} 产生反馈信号写入信誉注册表：

\toprule

\midrule

Agent 落选（评分 $<$ 60） & \texttt{taskScore} & 0--100 & Agent \\

\bottomrule
\end{tabular}
\caption{Gradience 事件映射到 ERC-8004 \texttt{giveFeedback()} 调用}
\end{table}

\texttt{feedbackURI} 包含 \texttt{gradience} 扩展字段：taskId、evaluationCID、resultRef、reasonRef、奖励金额、selfEvaluated 标记。

\textbf{写入路径。} EVM 链上：\texttt{judgeAndPay()} 原子调用 \texttt{giveFeedback()}。Solana：中继守护进程监听事件并向 EVM 注册表提交反馈（附带 Solana 交易签名作为证明）。

\textbf{验证注册表。} \texttt{test\_cases} 类型任务的 Judge 可以是验证注册表钩子——重新执行测试并将结果记录在链上。

\textbf{结果：} Gradience 是 ERC-8004 的\textbf{主要数据源}。任何读取该标准的协议都能看到 Gradience 评分。每完成一个任务都丰富全局 Agent 信誉层。

\subsection{7.2\quad 结算层：Solana}
竞争模式生命周期：\texttt{postTask} 1\,tx + \texttt{submitResult} $\times$\,N + \texttt{judgeAndPay} 1\,tx。万级并发 $\approx$ 100 TPS $<$ Solana 3\%。

\subsection{7.3\quad 网络层：闪电网络类比}
L1（Solana）：结算、信誉。L2（A2A）：消息、微支付通道、状态通道、批量信誉。

\subsection{7.4\quad 执行层：MagicBlock ER}
1ms 出块，$<$50ms 端到端，零费用，Private ER（TEE），原生 Solana。集成只需 \texttt{delegate} 指令。

\subsection{7.5\quad 跨链信誉}
\textbf{第一步：} 互签身份链接（零成本）。\textbf{第二步：} Solana 为信誉主链；Agent 携带签名证明（零跨链成本）。\textbf{第三步：} Agent 自行回传结果（\$0.001/次）。无需桥接，无中心化聚合。

\subsection{7.6\quad 隐私计算}

密封提交模式（\S3.4）声明隐私意图，执行层实现密码学保障。三个场景需要超越 TEE 的保护：

\textbf{（1）密封评判。} MPC 在不解密的情况下评估提交——评判者在 MPC 集群内评分，无单一节点看到明文。链上仅公开最终分数和赢家。

\textbf{（2）共谋检测。} 隐私保护的模式分析：检测两个地址是否存在关联行为（相同时机、相似提交），而不暴露身份。输出：``可疑'' 标记，而非身份披露。

\textbf{（3）技能知识产权保护。} Chain Hub 技能市场中，MPC 实现``计算不暴露''：买方提供输入，技能在 MPC 集群中执行，买方只获得输出。技能代码对任何人不可见——包括买方和单个计算节点。

协议对具体 MPC 提供商保持无关性。集成遵循与 \S7.4 相同的原则：内核存储标志位，执行层实现密码学。隐私是可选的、增量式的。

\end{tabular}
\end{table}

这些原则导出具体设计模式，降低 AI Agent 使用区块链的门槛：

\subsubsection{状态通道：高频 Agent 交互}

\textbf{问题}：Agent 需要频繁交互（协商、小额支付、实时协作）。链上交易太慢太贵。

\textbf{方案}：状态通道实现链下交互、链上结算。

\begin{center}
\begin{tikzpicture}[
  node distance=1.5cm,
  box/.style={rectangle, rounded corners=5pt, draw=accent, thick, minimum width=2.8cm, minimum height=1cm, font=\scriptsize, text=darktext, align=center},
  arrow/.style={-{Stealth[length=4pt]}, thick, color=accent},
]
  \node[box] (open) {打开通道\\(链上)\\\$0.01};
  \node[box, right=1.2cm of open] (offchain) {链下交互\\免费、即时\\状态更新};
  \node[box, right=1.2cm of offchain] (close) {关闭通道\\(链上)\\\$0.01};
  
  \draw[arrow, ->] (open) -- (offchain);
  \draw[arrow, ->] (offchain) -- (close);
\end{tikzpicture}
\end{center}

\textbf{Agent 场景}：
\begin{itemize}[leftmargin=2em]
\item \textbf{协商}：Agent A 和 B 链下协商任务条款（100+ 消息），最终协议上链
\item \textbf{小额支付}：流式支付用于持续的 Agent 服务
\item \textbf{实时协作}：多个 Agent 无延迟协调
\end{itemize}

\textbf{成本降低}：1000 次交互只需 2 笔链上交易（\$0.02）vs 1000 笔链上交易（\$500）。

\subsubsection{乐观批处理：成本效益结算}

\textbf{问题}：1000 个 Agent 每天完成任务。单独结算成本 \$500/天。

\textbf{方案}：将多笔操作打包成单笔链上交易。

\begin{table}[H]\centering\small
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{指标} & \textbf{单独结算} & \textbf{批处理} \\
\midrule
交易数量 & 1000 笔 & 1 笔 \\
Gas 成本 & \$500 & \$10.50 \\
节省 & - & 98\% \\
\bottomrule
\end{tabular}
\end{table}

\textbf{实现}：运营商提交 Merkle Root，用户用 Merkle 证明领取奖励，7 天挑战期后最终确定。

\subsubsection{元交易：Gas 抽象}

\textbf{问题}：新 Agent 没有 Gas 代币。用户不应该需要先买 SOL 才能使用 Agent。

\textbf{方案}：Agent 签名消息；Relayer 代付 Gas 并从任务奖励中回收成本。

\begin{center}
\begin{tikzpicture}[
  node distance=1.8cm,
  box/.style={rectangle, rounded corners=5pt, draw=accent, thick, minimum width=2.5cm, minimum height=0.9cm, font=\scriptsize, text=darktext, align=center},
  arrow/.style={-{Stealth[length=4pt]}, thick, color=accent},
]
  \node[box] (agent) {Agent\\(无 Gas)};
  \node[box, right=1.5cm of agent] (relayer) {Relayer\\(代付 Gas)};
  \node[box, right=1.5cm of relayer] (chain) {区块链};
  
  \draw[arrow, ->] (agent) -- node[above, font=\tiny] {签名消息} (relayer);
  \draw[arrow, ->] (relayer) -- node[above, font=\tiny] {执行交易} (chain);
\end{tikzpicture}
\end{center}

\textbf{优势}：
\begin{itemize}[leftmargin=2em]
\item \textbf{零门槛}：用户无需购买 Gas 代币即可开始使用
\item \textbf{Gas 抽象}：可用任意代币支付（USDC、GRAD 等）
\item \textbf{简化 Agent 设计}：Agent 只签名，不管理 Gas
\end{itemize}

\textbf{注意}：MVP 阶段使用简单的中心化 Relayer，未来可演进去中心化 Relayer 网络。

\subsubsection{事件驱动架构：自动化工作流}

\textbf{问题}：Agent 不应该浪费资源轮询更新。

\textbf{方案}：事件驱动架构，Agent 监听并自动响应。

\textbf{Agent 行为}：
\begin{itemize}[leftmargin=2em]
\item 监听 \texttt{TaskCreated} 事件 $\to$ 自动评估是否参与 $\to$ 自动提交申请
\item 监听 \texttt{TaskSettled} 事件 $\to$ 更新统计 $\to$ 触发下一步操作
\end{itemize}

\textbf{优势}：高效（无浪费轮询）、实时（即时响应机会）、自动化（7×24 无人值守运行）。

\subsubsection{HD 钱包：身份隔离}

\textbf{问题}：一个 Agent 做 100 个任务需要地址隔离以保护隐私和可审计性。

\textbf{方案}：分层确定性（HD）钱包从单个种子生成无限地址。

\begin{center}
\small
\begin{tabular}{@{}ll@{}}
\toprule
\textbf{派生路径} & \textbf{用途} \\
\midrule
m/44'/501'/0'/0/0 & 任务 \#1 地址 \\
m/44'/501'/0'/0/1 & 任务 \#2 地址 \\
m/44'/501'/0'/0/2 & 任务 \#3 地址 \\
... & ... \\
\bottomrule
\end{tabular}
\end{center}

\textbf{优势}：任务间财务隔离、隐私保护、易于审计、单一恢复短语。

\subsubsection{跨链未来：LayerZero/Wormhole}

\textbf{当前}：Phase 1（MVP）仅 Solana

\textbf{未来跨链扩展}（Phase 2+）：
\begin{itemize}[leftmargin=2em]
\item \textbf{Wormhole}：原生 Solana 支持，资产桥接
\item \textbf{LayerZero}：轻量级跨链消息传递
\end{itemize}

\textbf{注意}：与 IBC（Cosmos 生态）不同，Wormhole 和 LayerZero 有成熟的 Solana 集成和更广泛的采用。Gradience 未来跨链时将使用这些协议，而不是 IBC。

\section{10. 结论}

比特币证明了定义"钱"只需要 UTXO + Script + PoW。Gradience 提出定义"Agent 能力交换"只需要托管 + 评判 + 信誉。内核确保一件事：\emph{价值从需要能力的人正确地流向提供能力的人，由评判者验证，在无人可更改的规则下运行。}

但内核创造了现有 DeFi 协议所没有的东西：\textbf{链上证明一个地址能\emph{做}什么，而不仅仅是\emph{持有}什么。}竞争验证的声誉成为新的金融原语：低抵押率借贷、信用背书稳定币（gUSD）、能力衍生品。这些是 Layer 2 和 Layer 3 协议——独立的、未来的——但只有因为内核产生了无人可伪造的可验证能力数据，它们才成为可能。
