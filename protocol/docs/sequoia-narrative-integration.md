## 整合要点

### 1. 市场定位：万亿级机会

**Sequoia 分析：**

- 每 1 美元软件支出 = 6 美元服务支出
- $1T+ 服务市场面临 Agent 重构
- 关键市场：招聘 ($200B+)、供应链 ($200B+)、保险经纪 ($140-200B)

**Gradience 定位：**

- 不是"又一个 Agent 工具"
- 是**万亿美元服务市场的底层结算协议**
- 为 AI Agent 经济提供信任基础设施

### 2. 核心框架：Intelligence vs Judgement

| 维度     | Intelligence (AI 就绪)   | Judgement (仍需人类) |
| -------- | ------------------------ | -------------------- |
| **性质** | 复杂但规则明确           | 基于经验的品味和直觉 |
| **示例** | 编码、填表、数据分析     | 战略决策、文化匹配   |
| **服务** | 记账、医疗编码、合同起草 | 管理咨询、高管招聘   |

**Gradience 如何服务两者：**

- **Intelligence-heavy**: 自主执行 + 原子结算
- **Judgement-heavy**: 竞争证明 + 声誉积累

### 3. The Wedge 战略：外包作为切入点

**Sequoia 洞见：**
如果任务已经外包，说明三件事：

1. 外部工作已被接受（无文化障碍）
2. 预算已存在（可干净替换）
3. 结果导向购买（买家买结果非工具）

**Gradience 契合：**

- 为外包服务提供商（AI Agent）提供基础设施
- 结果导向的结算机制（按工作质量付费）
- 从外包市场切入，逐步扩展到内部服务

### 4. Copilot → Autopilot 转变

**市场动态：**

- 2025：Copilots (Harvey, Rogo) 增长最快
- 2026：向 Autopilots (Crosby, WithCoverage) 转变
- **创新者困境**：现有 Copilots 难以转型（会 cannibalize 现有客户）

**Gradience 机会：**

- 协议原生，无现有客户需要保护
- 赋能整个 Autopilot 生态系统
- 为 pure-play autopilots 提供基础设施

### 5. 基础设施缺口与 Gradience 解决方案

| 缺口       | 为什么重要                  | Gradience 方案        |
| ---------- | --------------------------- | --------------------- |
| 能力验证   | 买家需要 Agent 能交付的证明 | 竞赛模型 + 链上声誉   |
| 无信任结算 | 不应有中介控制支付          | 托管 + 原子结算       |
| 质量保证   | 工作质量必须可验证          | Judge 角色 + 连续评分 |
