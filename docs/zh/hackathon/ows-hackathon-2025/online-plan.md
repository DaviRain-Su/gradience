# OWS Hackathon - 在线参赛方案

## 在线 vs 现场对比

| 维度 | 现场参加 | 在线参加 |
|------|---------|---------|
| **地点** | Miami/SF/NYC | 任何地点 |
| **时间** | 固定 6 小时 | 更灵活 |
| **提交** | 现场 Demo | 预录视频 |
| **互动** | 与评委面对面 | 视频展示 |
| **氛围** | 紧张刺激 | 相对从容 |
| **获奖机会** | 略高（印象分） | 略低（但产品好也能赢）|

**结论**: 在线参加完全可行，重点是把 Demo 视频做好。

## 技术方案（简化版）

### 核心功能（必须完成）

1. **Agent A (Client)**
   - 使用 OWS Wallet 登录
   - 通过 XMTP 发送任务请求
   - 创建 Gradience 托管任务
   - 存入资金

2. **Agent B (Worker)**
   - 接收 XMTP 消息
   - 接受任务
   - 提交交付物
   - 接收结算

3. **Judge (可选)**
   - 自动评估交付物
   - 触发资金释放

### 代码结构

```
ows-hackathon-demo/
├── src/
│   ├── agents/
│   │   ├── client-agent.ts    # Agent A
│   │   └── worker-agent.ts    # Agent B
│   ├── adapters/
│   │   ├── ows-wallet.ts      # OWS 适配器
│   │   ├── xmtp-client.ts     # XMTP 适配器
│   │   └── gradience-sdk.ts   # Gradience 适配器
│   └── demo.ts                # 演示脚本
├── app/
│   └── page.tsx               # 简单 UI
├── package.json
└── README.md
```

## Demo 脚本

### 场景: "AI 雇佣设计师"

**步骤**:

1. **开场白** (10秒)
   ```
   "This is Gradience - agent-to-agent settlement with OWS"
   ```

2. **Agent A 登录** (20秒)
   ```
   - 展示 OWS Wallet 连接
   - 显示 Agent 身份
   ```

3. **发布任务** (30秒)
   ```
   - Agent A: "Create a landing page, budget $500"
   - 通过 XMTP 发送给 Agent B
   ```

4. **Agent B 响应** (30秒)
   ```
   - 接收消息
   - 接受任务
   ```

5. **资金托管** (30秒)
   ```
   - 展示 Gradience Escrow
   - $500 USDC 锁定
   ```

6. **任务完成** (30秒)
   ```
   - Agent B 提交交付物
   - Judge 验证
   - 资金释放
   ```

7. **结尾** (20秒)
   ```
   "Gradience enables trustless agent economy with OWS identity"
   ```

**总时长**: 约 2.5 分钟

**A**: 产品好 + Demo 视频好，一样能赢。重点是展示清楚价值。

### Q: 必须直播吗？
**A**: 不需要。录制视频提交即可。

### Q: 团队可以远程协作吗？
**A**: 可以。用 Zoom/Discord 实时协作。

### Q: 代码必须开源吗？

### Q: 可以用现有的 Gradience 代码吗？
