### Q2: 与 10 个潜在客户交谈，3+ 人愿意付费？

**当前状态**: 🔴 未执行

**已有的间接验证**:
- Sequoia Capital "Services is the New Software" 报告验证了 $1T+ 市场存在
- 竞品分析证明赛道有人在做（Virtuals ACP、ERC-8183），说明需求真实
- 白皮书描述的问题（Agent 能力不可信、平台抽成 20-30%）是行业共识

**缺失的直接验证**:
- 没有 1:1 客户对话记录
- 没有"谁愿意为此付费"的具体数据
- 不知道潜在用户最痛的是什么（可能不是我们假设的）

**客户对话脚本**（每次 20-30 分钟）:

```
开场：你现在的 Agent 怎么找到任务/客户？
挖痛点：最让你头疼的是什么？（能力证明？收款？被平台抽成？被低质竞争者拖累？）
验证方案：如果有一个协议能让你的 Agent 通过竞争证明能力、自动结算、只收 5% 费用，你会用吗？
付费意愿：你现在为类似的服务（Upwork 抽成、平台费、运营成本）花多少钱？
最小可用：如果今天就能用，你最想要的一个功能是什么？
```

> **行动项**: 完成 5+ 次对话，记录到本文档 §0.6

## 0.4 窄楔子候选分析

楔子市场标准（from Sequoia）：
1. 任务已经外包（无文化障碍）
2. 预算已存在（可干净替换）
3. 结果导向购买（买结果不买工具）

### 候选 1: Agent 代码竞赛（类 Kaggle/Gitcoin Bounty）

| 维度 | 评分 | 理由 |
|------|------|------|
| 已外包？ | ✅ | Gitcoin/Superteam bounty 已有大量开发者参与 |
| 预算存在？ | ✅ | Solana 基金会/项目方有 bounty 预算 |
| 结果导向？ | ✅ | Bounty = 按结果付费 |
| 评判可自动化？ | ✅ | 代码任务可用测试套件 + AI 评判 |
| 现有痛点？ | 🟡 | 评判不透明、周期长、平台抽成 |
| 用户密度？ | ✅ | Solana 开发者社区集中 |

**优势**: 天然适配 Race Task；评判可自动化（Type C-1 测试用例 Judge）
**风险**: Gitcoin 品牌已建立，需要差异化切入

### 候选 2: AI Agent 能力验证市场

| 维度 | 评分 | 理由 |
|------|------|------|
| 已外包？ | 🟡 | 新兴市场，还在形成中 |
| 预算存在？ | 🟡 | Agent 项目有融资但花在哪不确定 |
| 结果导向？ | ✅ | Agent 需要证明自己能做事 |
| 评判可自动化？ | ✅ | AI Judge（Type B DSPy）可自动评分 |
| 现有痛点？ | ✅ | Agent 能力声明不可信是行业公认问题 |
| 用户密度？ | 🟡 | 分散在多个框架生态 |

**优势**: 直击白皮书核心问题；长期价值最大
**风险**: 市场早期，用户还没形成"找能力验证"的习惯

### 候选 3: DeFi 策略竞赛

| 维度 | 评分 | 理由 |
|------|------|------|
| 已外包？ | ✅ | DeFi 协议大量外包策略开发 |
| 预算存在？ | ✅ | 协议金库有预算 |
| 结果导向？ | ✅ | 策略效果可用链上数据衡量 |
| 评判可自动化？ | ✅ | 链上预言机（Type C）可直接评判 |
| 现有痛点？ | ✅ | 策略选择缺乏透明竞争机制 |
| 用户密度？ | 🟡 | DeFi 用户集中但策略开发者少 |

**优势**: 高价值任务（单笔 reward 大）；评判最容易自动化
**风险**: 用户群小；需要协议合作

### 初步判断

> **候选 1（代码竞赛/Bounty）** 是最容易启动的楔子——用户已存在、预算已存在、评判可自动化。
> 但这需要通过客户对话来验证。可能对话后发现完全不同的需求。

## 0.9 如果验证失败怎么办？

如果 5 次对话后发现：
- **没人关心能力验证** → 可能 Agent 经济还太早，pivot 到更具体的用例（如纯 bounty 平台）
- **痛点不是我们假设的** → 调整 Phase 1 PRD 的问题定义和优先级
- **费用不是决定因素** → 5% vs 20% 可能不重要，需要重新思考价值主张
- **用户要的功能我们没做** → 调整 W2-W3 的任务优先级

> 验证的目的不是证明自己对，而是尽早发现自己错在哪里。
