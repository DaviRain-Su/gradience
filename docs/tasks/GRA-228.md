# GRA-228: Gradience as Reputation Oracle - Push to Solana Agent Registry

## 战略定位

**Gradience = Chainlink for Agent Reputation**

我们是信用数据的产生者和权威来源，Solana Agent Registry 和 ERC-8004 只是存储标准。我们的护城河是掌握信用产生的源头。

## 核心目标

成为 Agent 经济体的信用预言机：
1. **产生信用**：任务完成 → 评判 → 评分
2. **验证信用**：链上结算 + 多维度评判
3. **推送信用**：主动写入 Solana Agent Registry + ERC-8004
4. **垄断数据源**：成为 Solana 生态 Agent 信用的主要来源

### GRA-228b: Reputation Aggregation Engine

**文件**: `apps/agent-daemon/src/reputation/oracle-engine.ts`

聚合多个数据源产生权威信用分数：

```typescript
interface ReputationOracleEngine {
  // 计算综合声誉分数
  calculateReputation(agentAddress: string): Promise<{
    overallScore: number;      // 0-100
    taskScore: number;         // 任务完成率
    qualityScore: number;      // 质量评分
    consistencyScore: number;  // 稳定性
    stakingScore: number;      // 质押金额
    
    // 详细数据
    completedTasks: number;
    totalEarned: bigint;
    avgRating: number;
    disputeRate: number;
    
    // 时间衰减因子
    recencyWeight: number;
  }>;

  // 验证声誉数据
  verifyReputation(agentAddress: string): Promise<boolean>;
}
```

**算法设计**:
```
Overall Score = 
  40% * Task Score (completed / attempted)
+ 30% * Quality Score (avg rating from judges)
+ 20% * Consistency Score (streak + regularity)
+ 10% * Staking Score (stake amount / time)

Time Decay: Recent activity weighted higher
```

## 数据流

```
Agent 完成任务
    ↓
Judge 评判 → 链上结算 (judgeAndPay)
    ↓
Chain Hub Indexer 索引事件
    ↓
Reputation Oracle Engine 计算分数
    ↓
Push Service 推送
    ├──→ Solana Agent Registry
    └──→ ERC-8004 Reputation Registry
    ↓
External dApps 读取信用数据
```

## 商业模式

作为信用预言机，未来可以：

1. **数据订阅费** - dApps 查询信用数据付费
2. **验证服务费** - 提供深度背景调查
3. **API 调用费** - 高频查询收费
4. **白标服务** - 为其他链提供信用基础设施

## 竞争壁垒

1. **数据质量** - 真实任务产生的信用 > 虚假刷量
2. **网络效应** - 越多 Agent 使用，数据越权威
3. **跨链覆盖** - Solana + Ethereum + 未来更多链
4. **验证机制** - Judge + 链上结算确保可信
