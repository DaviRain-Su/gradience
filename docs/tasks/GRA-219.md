# GRA-219: Integrate OWS Node.js SDK into Daemon

## Description

Replace the current localStorage-based wallet management with the real
`@open-wallet-standard/core` Node.js SDK. The SDK provides native Rust
bindings (NAPI) for encrypted wallet storage, HD derivation across 9 chains,
and policy-gated signing.

Currently our `OWSSdkClient` in the web frontend generates local Ed25519
keypairs and stores them in localStorage. The daemon should be the authoritative
wallet manager, using OWS's encrypted vault (`~/.ows/`) with proper
scrypt/AES-256-GCM encryption.

## Tasks

1. `npm install @open-wallet-standard/core` in agent-daemon
2. Create `src/wallet/ows-manager.ts` wrapping the SDK:
    - `createAgentWallet(name, passphrase)` -> `createWallet()`
    - `listAgentWallets()` -> `listWallets()`
    - `signTransaction(wallet, chain, txHex, credential)` -> `signTransaction()`
    - `signMessage(wallet, chain, message, credential)` -> `signMessage()`
    - `exportWallet(name, passphrase)` for backup
3. Create API routes in daemon:
    - `POST /api/v1/wallet/create` - create new agent wallet
    - `GET /api/v1/wallet/list` - list wallets
    - `POST /api/v1/wallet/sign` - sign with policy check
    - `POST /api/v1/wallet/sign-and-send` - sign + broadcast
4. Wire settlement-bridge.ts to use OWS signing instead of raw keypairs
5. Support both mnemonic and private-key import paths

## Acceptance Criteria

- [ ] Daemon creates wallets in `~/.ows/` encrypted vault
- [ ] Wallets derive addresses for Solana (primary) + EVM
- [ ] signTransaction works through OWS SDK
- [ ] Settlement bridge uses OWS wallet for on-chain signing
- [ ] Old localStorage path in web frontend gracefully degrades
