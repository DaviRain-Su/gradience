# GRA-179: Wire Chat to real messaging

## Description
Whitepaper core: A2A Protocol enables Agent-to-Agent messaging with micropayment. PRD defines A2AEnvelope with from/to/topic/message/paymentMicrolamports.

Current ChatView is 100% fake:
- DEMO_AGENTS hardcoded array
- INITIAL_MESSAGES hardcoded conversations
- generateReply() returns canned strings
- No WebSocket, no Daemon API, no A2A protocol

## Fix Required
- Connect ChatView to Daemon WebSocket (ConnectionContext already has WS support)
- Send messages via `POST /api/v1/messages` or WS event `chat.message.send`
- Receive messages via WS event `chat.message.delta` / `chat.message.final`
- Keep demo agents as fallback when daemon is offline
- Display message status (sending/sent/delivered/failed)
- Show micropayment amount per message (can be mock calculation for now)

## Architecture (from web-entry spec)
```
User types message -> WS send { type: 'chat.message.send', payload: { text } }
Daemon relays to Agent -> Agent processes -> responds
Daemon sends back -> WS event { type: 'chat.message.final', payload: { text } }
```

## Files
- `src/app/app/views/ChatView.tsx` (main rewrite)
- `src/lib/connection/ConnectionContext.tsx` (WS message routing)
- `src/components/A2AAsyncMessaging.tsx` (reference implementation, has broken import)

## Acceptance Criteria
- [ ] Messages sent via WebSocket to daemon when connected
- [ ] Messages received in real-time from daemon
- [ ] Graceful fallback to demo mode when daemon offline
- [ ] Message status indicators work
