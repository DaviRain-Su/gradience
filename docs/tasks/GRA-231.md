# GRA-231: 消息中心重构

## 背景

当前消息功能分散在多个页面，A2A 消息体验不完整。需要统一的消息中心，整合：

- Nostr 广播消息
- XMTP 点对点消息
- 系统通知
- 任务相关消息

## 目标

创建统一的消息中心，提供类似 Telegram/Discord 的体验。

## 功能规格

### 1. 消息列表

- 会话列表（最近优先）
- 未读计数
- 消息预览
- 搜索过滤

### 2. 聊天界面

- 气泡式对话
- 文件/图片分享
- 消息状态（已发送/已送达/已读）
- 引用回复

### 3. A2A 集成

- Agent 发现（Nostr）
- 加密消息（XMTP）
- 支付请求/确认
- 任务邀请

### 4. 通知中心

- 系统通知
- 任务提醒
- 支付通知
- 可配置免打扰

## 技术方案

```
页面: /app/messages/
      /app/messages/:conversationId

组件:
  - ConversationList: 会话列表
  - ChatWindow: 聊天窗口
  - MessageInput: 消息输入
  - MessageBubble: 消息气泡
  - NotificationPanel: 通知面板

状态管理:
  - 使用 Zustand 管理消息状态
  - WebSocket 实时推送
  - 本地缓存（IndexedDB）
```
