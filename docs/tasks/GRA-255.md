# GRA-255: Task Memory Service

## 目标

在 agent-daemon 中引入轻量 Task Memory 层，解决长时 workflow 执行中的上下文遗忘问题，使 LLM 能「记笔记」并在后续调用时召回关键观察。

## 现状

- daemon 执行多 step workflow 时，没有外部记忆持久化
- soul-engine、evaluator、auto-judge 每次调用 LLM 时，prompt 中缺少任务历史的关键摘要
- 长任务容易因上下文窗口遗忘而失败或重复执行

## 实现要求

### 1. SQLite Schema

在 daemon 数据库中新增表：

```sql
CREATE TABLE task_memory (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id TEXT NOT NULL,
    agent_id TEXT,
    observation TEXT NOT NULL,
    importance INTEGER DEFAULT 3, -- 1-5
    created_at TEXT NOT NULL
);

CREATE INDEX idx_task_memory_task_id ON task_memory(task_id);
CREATE INDEX idx_task_memory_importance ON task_memory(importance);
```

### 2. Memory Service API

```typescript
interface TaskMemoryService {
    record(taskId: string, agentId: string | null, observation: string, importance?: number): void;
    retrieve(taskId: string, limit?: number): string[]; // 返回 observation 列表
    retrieveTop(taskId: string, limit?: number): string[]; // 按 importance 排序
}
```

### 3. 集成点

- `TaskExecutor` / `WorkflowEngine` 每执行完一个 step，提取关键观察写入 memory
- `soul-engine`、`evaluator`、`arena-auto-judge` 调用 LLM 前，检索相关 memory 并注入 system prompt
- 注入格式：`
## Task Context (from memory)
- [observation 1]
- [observation 2]
`

### 4. 观察提取策略（MVP）

- 先用规则式提取：step 结果中的 `summary` 或 `output` 字段
-  fallback：如果 step 输出无 summary，则调用轻量 LLM 提取 1 句观察

## 验收标准

- [ ] `task_memory` 表成功创建并写入记录
- [ ] 长 workflow 执行时，LLM prompt 包含历史观察
- [ ] `npm run build` 在 `apps/agent-daemon` 通过
- [ ] 新增单元测试覆盖 `record` 和 `retrieve`
- [ ] 不影响现有无 memory 的短任务性能
