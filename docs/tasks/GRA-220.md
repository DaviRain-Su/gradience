# GRA-220: Implement Policy Engine for Agent Signing

## Description

The OWS Policy Engine is the core of agent autonomy. A master wallet owner
defines policies (chain restrictions, spending limits, time bounds, custom
executable checks) and attaches them to API keys. Each agent gets a token
that both authenticates and decrypts its wallet scope -- with policy checks
enforced before any key material is touched.

Currently our `AgentSigningPolicy` in `agent-router.ts` has
`dailyLimitUsd` and `requireMasterApprovalAboveUsd` fields but they are
never enforced. We need to implement real policy evaluation.

## Tasks

1. Create `src/wallet/policy-engine.ts`:
    - Parse OWS policy JSON format (rules + executable)
    - Implement declarative rules: `allowed_chains`, `expires_at`
    - Implement custom executable evaluation (spawn + stdin/stdout JSON)
    - AND semantics: all policies must allow
    - Default-deny on failure/timeout (5s)
2. Create built-in policies for Gradience:
    - `solana-devnet-only`: restrict to `solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1`
    - `daily-spend-limit`: custom executable checking SOL spending
    - `task-settlement-only`: only allow Arena program interactions
3. Create `PolicyContext` builder from transaction data
4. Integrate with signing flow: policy check before `signTransaction()`
5. Store policies in `~/.ows/policies/` per OWS spec

## Acceptance Criteria

- [ ] Policies stored as JSON in `~/.ows/policies/`
- [ ] `allowed_chains` rule blocks signing on wrong chain
- [ ] `expires_at` rule blocks expired API keys
- [ ] Custom executable policies work (stdin JSON, stdout result)
- [ ] Default-deny on timeout/error
- [ ] Policy evaluation logged for audit
