# GRA-234: CI/CD 完整流水线

## 背景

当前部署流程：
- Vercel 未连接 Git，需手动部署
- 缺乏自动化测试
- 无预览环境

## 目标

建立完整的 CI/CD 流水线。

## 流水线设计

### 1. Pull Request 流程
```
PR 创建
  ↓
GitHub Actions:
  - Lint (ESLint + Rust fmt)
  - Type Check
  - Unit Tests
  - Build Test
  - Vercel Preview Deploy
  ↓
Code Review
  ↓
Merge to main
```

### 2. Main 分支部署
```
Push to main
  ↓
GitHub Actions:
  - Full Test Suite
  - Build
  - Deploy to Staging
  - E2E Tests
  - Deploy to Production
```

### 3. 配置

#### Vercel
- 连接 GitHub Repo
- 启用自动部署
- 配置环境变量

#### GitHub Actions
```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  lint:
  typecheck:
  test:
  build:
```
