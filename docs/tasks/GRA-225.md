# GRA-225: Reputation Wallet - Connect Reputation to OWS Wallet Creation

## Description

Currently, OWS Wallet creation and Reputation are two separate systems:
- OWS Wallet: Created via daemon with policy-based signing
- Reputation: Fetched from Chain Hub indexer, displayed in UI

**The Gap**: When creating an Agent Wallet, the system does NOT automatically fetch the agent's reputation and apply it to set the initial policy. This breaks the "Reputation Wallet" concept where reputation should drive wallet permissions.

## Goals

Implement the **Reputation Wallet** concept:
1. **Master Wallet Reputation** = Aggregate of all Agent Wallet reputations
2. **Auto-Policy Assignment**: Creating Agent Wallet → Fetch reputation → Auto-set policy
3. **Reputation-Policy Sync**: Reputation changes → Policy updates automatically

## Implementation Tasks

### Daemon Side (GRA-225a)

1. **Integrate Chain Hub into Wallet Creation**
   - File: `apps/agent-daemon/src/wallet/ows-wallet-manager.ts`
   - When `createWallet()` is called:
     ```typescript
     // Fetch reputation from Chain Hub
     const reputation = await chainHub.getReputation(agentAddress);
     
     // Calculate policy based on reputation score
     const policy = calculatePolicyFromReputation(reputation.score);
     
     // Store reputation + policy with wallet
     await walletStore.create({
       ...walletData,
       reputationScore: reputation.score,
       policy,
     });
     ```

2. **Add Reputation Aggregation API**
   - File: `apps/agent-daemon/src/api/routes/ows.ts`
   - New endpoint: `GET /api/v1/ows/wallets/:id/reputation`
   - Returns aggregated reputation for master wallet + all sub-wallets

3. **Reputation-Based Policy Engine**
   - File: `apps/agent-daemon/src/wallet/reputation-policy.ts` (new)
   - Map reputation score to policy:
     - 0-30 (Bronze): $3-30 daily, manual approval, Solana only
     - 31-50 (Silver): $310-500 daily, manual approval
     - 51-80 (Gold): $510-800 daily, auto-approval
     - 81-100 (Platinum): $810-1000 daily, auto-approval, all chains

### Web Side (GRA-225b)

1. **Update useOWSAgentRouter**
   - When creating sub-wallet, pass agent address for reputation lookup
   - Display reputation-derived policy in wallet list

2. **Update PolicyManager**
   - Show "Reputation-Derived Policy" badge
   - Display reputation score next to each wallet
   - Allow manual override with warning

3. **Create useAggregatedReputation Hook**
   - Fetch all agent wallet reputations under master wallet
   - Calculate aggregate score
   - Display in My Agent tab

## Acceptance Criteria

- [ ] Creating Agent Wallet auto-fetches reputation from Chain Hub

- [ ] Master wallet shows aggregated reputation from all agents
- [ ] Policy updates when reputation changes (manual refresh or webhook)

## API Changes

```typescript
// GET /api/v1/ows/wallets/:id/reputation
interface WalletReputationResponse {
  walletId: string;
  agentAddress: string;
  reputationScore: number;
  tier: 'bronze' | 'silver' | 'gold' | 'platinum';
  policy: {
    dailyLimitUsd: number;
    requireApprovalAboveUsd: number;
    allowedChains: string[];
  };
  updatedAt: string;
}

// GET /api/v1/ows/wallets/master/:address/aggregate-reputation
interface AggregateReputationResponse {
  masterWallet: string;
  aggregateScore: number; // weighted average
  agentCount: number;
  agents: Array<{
    walletId: string;
    address: string;
    reputationScore: number;
    weight: number;
  }>;
}
```

## Notes

The "Reputation Wallet" concept is:
1. **Identity**: Master wallet as identity anchor
2. **Agents**: Multiple sub-wallets with individual reputations
3. **Aggregation**: Master reputation = f(agent reputations)
4. **Policy**: Wallet permissions derived from reputation
5. **Trust**: External parties trust based on aggregated reputation
