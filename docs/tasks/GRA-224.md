# GRA-224: Policy Management UI

## Description

Add a Policy management section to the My Agent tab where wallet owners
can create, view, and manage signing policies for their agent sub-wallets.

## Tasks

1. Policy list view:
    - Fetch policies from daemon `GET /api/v1/policies`
    - Show name, rules summary, attached keys count
2. Create policy form:
    - Chain restriction (checkbox: Solana devnet, mainnet, EVM chains)
    - Expiry date picker
    - Daily spending limit (SOL amount)
    - Task-only mode (restrict to Arena program)
3. Attach policy to API key:
    - When creating sub-wallet, select policies to attach
    - Show which policies are active per agent
4. Policy audit log:
    - Show recent policy evaluations (allow/deny)
    - Show denial reasons
5. Pre-built policy templates:
    - "Devnet Only" - safe for testing
    - "Conservative Agent" - 0.1 SOL daily, Solana only, 30 day expiry
    - "Task Runner" - Arena program only, 1 SOL daily

## Acceptance Criteria

- [ ] Can create policies with chain + time + spend rules
- [ ] Can attach policies to API keys
- [ ] Policy templates available as one-click setup
- [ ] Active policies visible per agent sub-wallet
- [ ] Denial log shows why transactions were blocked

## Dependencies

- GRA-220 (daemon policy engine)
- GRA-221 (API key management)
