# GRA-235: 核心模块测试覆盖

## 背景

当前测试覆盖率低，关键模块缺乏测试。

## 目标

为核心模块建立测试覆盖。

## 测试策略

### 1. 单元测试

| 模块              | 优先级 | 关键测试点     |
| ----------------- | ------ | -------------- |
| Evaluator         | P0     | Judge 评分逻辑 |
| Revenue Engine    | P0     | 收益计算       |
| Reputation Oracle | P0     | 分数计算       |
| Payment Service   | P0     | 支付流程       |

### 2. 集成测试

- Daemon API 端点
- 链上交互（mock）
- A2A 消息流程

### 3. E2E 测试

- 用户注册流程
- 任务发布-申请-提交-评判流程
- 支付流程

## 技术栈

```
Frontend: Vitest + Testing Library + MSW
Backend:  Node:test + Supertest
E2E:      Playwright
```

## 验收标准

- [ ] 核心模块单元测试 > 80% 覆盖
- [ ] 集成测试覆盖主要 API
- [ ] E2E 测试覆盖关键用户流程
- [ ] CI 中运行测试
