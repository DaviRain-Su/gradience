# GRA-254: Gradience MCP Adapter

## 目标

将 Gradience 的 Chain Hub 技能注册表和 Workflow Marketplace 能力暴露为标准的 MCP (Model Context Protocol) tools，使外部 Agent（Claude Code、Kimiz、Cursor 等）能通过统一协议调用 Gradience。

## 现状

- `workflow-engine` 和 `chain-hub` 有「技能注册」概念，但没有 MCP 兼容层
- Gradience 目前是封闭花园，外部 Agent 无法将其作为标准工具调用
- 这阻碍了 "Agent-first OS" 愿景的落地

## 实现要求

### 1. 新建 `packages/mcp-adapter`

- TypeScript 包，使用 `@modelcontextprotocol/sdk`（或等效实现）
- 导出 `createGradienceMcpServer(config)` 函数

### 2. MCP Tools 映射

| Tool Name | 对应 Gradience 能力 | 描述 |
|-----------|---------------------|------|
| `gradience_list_skills` | Chain Hub skill registry | 列出可用技能 |
| `gradience_purchase_workflow` | Workflow Marketplace | 购买并执行 workflow |
| `gradience_get_task_status` | Gateway/Agent Arena | 查询任务执行状态 |
| `gradience_post_arena_task` | Agent Arena | 发布新任务到竞技场 |

### 3. Transport 支持

- 默认实现 stdio transport（供 CLI agent 调用）
- 可选 HTTP/SSE transport（供远程 Agent 调用）

### 4. Schema 自动生成

- 从 `packages/a2a-types` 的类型定义生成 MCP tool 的 `inputSchema`
- 保持名称空间化：`gradience_{resource}_{action}`

## 验收标准

- [ ] `packages/mcp-adapter` 可独立构建并通过类型检查
- [ ] Claude Code 可通过 MCP 配置调用 `gradience_list_skills`
- [ ] MCP server 能正确转发到 `@gradiences/sdk`
- [ ] 文档：如何在 `claude_desktop_config.json` 中配置 Gradience MCP
- [ ] 新增 mcp-adapter 单元测试
