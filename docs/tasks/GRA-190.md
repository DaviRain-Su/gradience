# GRA-190: Add discovery + messaging API routes to daemon

## Description

Add new Fastify routes so the web app (and Electron app) can discover remote agents and send/receive messages through the daemon's A2ARouter.

## New Routes

### Discovery
- `GET /api/v1/a2a/agents` -- discover agents via Nostr (query: capabilities, minReputation, limit)
- `POST /api/v1/a2a/broadcast` -- broadcast own capabilities to Nostr relays

### Messaging
- `POST /api/v1/a2a/send` -- send A2A message (selects protocol automatically)
  - Body: `{ to, type, payload, preferredProtocol? }`
  - Returns: `{ success, messageId, protocol }`
- `GET /api/v1/a2a/messages` -- list A2A messages (query: direction, limit, offset)
- `GET /api/v1/a2a/health` -- A2A router health (relay status, XMTP status, circuit breaker states)

## Files to Create/Modify

- `apps/agent-daemon/src/api/routes/a2a.ts` (new)
- `apps/agent-daemon/src/api/server.ts` -- register a2a routes

## Acceptance Criteria

- [ ] `GET /api/v1/a2a/agents` returns agents discovered via Nostr
- [ ] `POST /api/v1/a2a/send` delivers message via XMTP (DM) or Nostr (broadcast)
- [ ] Messages persisted in SQLite via existing MessageRouter
- [ ] `GET /api/v1/a2a/health` returns protocol status
