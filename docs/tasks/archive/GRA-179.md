# GRA-179: [EVM] Integrate JudgeRegistry into AgentArenaEVM

## Description

The EVM-side `AgentArenaEVM.sol` contract currently lacks a programmable judge assignment mechanism. This task wires the `JudgeRegistry` into `AgentArenaEVM` so that tasks posted without a designated judge can be automatically assigned from a staked judge pool.

Key behaviors:
- `postTask(judge = address(0))` triggers automatic judge selection via `JudgeRegistry`.
- `judgeAndPay()` verifies that the caller is the assigned judge.
- `reassignJudge()` allows anyone to reallocate a judge after the `REASSIGN_WINDOW` if the original judge has not acted, slashing a portion of their stake.

## Implementation Plan

### 1. Interface Definition
- Create `src/interfaces/IJudgeRegistry.sol` (pragma `^0.8.24`).
- Expose `selectJudge(uint8 category, uint256 randomness)` and `slash(address judge, uint256 amount)`.

### 2. AgentArenaEVM Updates
- Add `address public judgeRegistry` state variable.
- Add `setJudgeRegistry(address)` setter (owner only).
- In `postTask()`:
  - If `judge == address(0)`, call `IJudgeRegistry(judgeRegistry).selectJudge(category, block.prevrandao)`.
  - For high-value tasks (`reward > PREVRANDAO_MAX_REWARD`), require a designated judge or VRF.
- In `judgeAndPay()`:
  - Add `require(msg.sender == task.judge, "NOT_ASSIGNED_JUDGE")`.
- Add `reassignJudge(uint256 taskId)`:
  - Callable after `task.judgeDeadline - REASSIGN_WINDOW` if `!hasJudged[taskId]`.
  - Slash 10% of the original judge's stake.
  - Assign a new judge via `selectJudge`.

### 3. Storage Layout Guard
- Ensure `Task` struct and mappings comply with the frozen layout from `pre-implementation-review.md`.

### 4. Foundry Tests
- `test/AgentArenaEVM_JudgeRegistry.t.sol`
  - Auto-assignment on `postTask`
  - Unauthorized `judgeAndPay` reverts
  - Reassignment and slash logic
  - High-value task designation rules

## Acceptance Criteria
