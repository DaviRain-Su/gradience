# GRA-191: Wire agentm-web ChatView + DiscoverView to daemon A2A API

## Description

Connect the web app's ChatView and DiscoverView to the daemon's new A2A API routes (from GRA-190). Currently ChatView uses demo data and DiscoverView shows hardcoded agents.

## Changes

### DiscoverView
- Call `GET /api/v1/a2a/agents` to fetch real agents from Nostr relays
- Merge with local demo agents as fallback
- Show discovery source badge (Nostr / Demo)

### ChatView
- Send messages via `POST /api/v1/a2a/send` (type: direct_message, preferredProtocol: xmtp)
- Poll `GET /api/v1/a2a/messages` for inbound messages (or use WebSocket if available)
- Show protocol indicator (XMTP E2E / Demo)

### ConnectionContext
- Add `a2aSend()` and `a2aDiscover()` helper methods that proxy to daemon API

## Files to Modify

- `apps/agentm-web/src/lib/connection/ConnectionContext.tsx`
- `apps/agentm-web/src/app/app/page.tsx` (DiscoverView section)
- `apps/agentm-web/src/app/app/views/ChatView.tsx`

## Acceptance Criteria

- [ ] DiscoverView shows agents from Nostr when daemon is connected
- [ ] ChatView sends messages through XMTP via daemon proxy
- [ ] Graceful fallback to demo data when daemon is offline
- [ ] Protocol source displayed in UI (Nostr / XMTP / Demo)
