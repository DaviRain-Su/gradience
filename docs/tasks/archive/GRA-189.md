# GRA-189: Integrate A2ARouter into agent-daemon

## Description

Replace the stub `A2ARouter` interface in `apps/agent-daemon/src/a2a-router/router.ts` with a real implementation that uses `@gradiences/nostr-adapter` for discovery and the existing `@gradiences/xmtp-adapter` for direct messaging.

The Daemon must:

1. Initialize A2ARouter on startup with Nostr + XMTP adapters
2. Wire inbound A2A messages into the existing `MessageRouter`
3. Route outbound messages through the A2A protocol selection (Nostr for broadcast/discovery, XMTP for DM)
4. Expose A2A health in the `/api/v1/status` endpoint

## Files to Modify

- `apps/agent-daemon/src/a2a-router/router.ts` -- replace stub with real class
- `apps/agent-daemon/src/daemon.ts` -- instantiate A2ARouter, wire lifecycle
- `apps/agent-daemon/src/config.ts` -- add nostr relay + xmtp config fields
- `apps/agent-daemon/src/api/server.ts` -- pass a2aRouter to deps
- `apps/agent-daemon/src/api/routes/status.ts` -- include A2A health
- `apps/agent-daemon/src/messages/message-router.ts` -- accept A2ARouter for outbound
- `apps/agent-daemon/package.json` -- add dependencies

## Acceptance Criteria

- [ ] Daemon starts with A2ARouter initialized (Nostr connects to 4 relays)
- [ ] `POST /api/v1/messages/send` routes through A2ARouter instead of ConnectionManager only
- [ ] Inbound Nostr presence events populate discoverable agents
- [ ] Inbound XMTP messages route to MessageRouter
- [ ] `/api/v1/status` includes `a2a: { nostr: { connected, relayCount }, xmtp: { connected } }`
- [ ] `agentd` builds and starts without errors
