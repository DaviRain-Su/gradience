# GRA-177: Fix DynamicLoginButton type errors

## Description

`DynamicLoginButton.tsx` uses `isAuthenticated` and `isLoading` from `useDynamicContext()`, but Dynamic SDK v4 does not expose these properties. This causes TypeScript errors and potential runtime crashes.

Also affects `lib/dynamic/provider.tsx` which uses the same non-existent properties.

## Current State

- `isAuthenticated` -> does not exist on `UseDynamicContext`
- `isLoading` -> does not exist, should use `sdkHasLoaded`
- Component renders in the app sidebar, so this is a blocking issue

## Fix Required

- Replace `isAuthenticated` with `!!user` (user is available on context)
- Replace `isLoading` with `!sdkHasLoaded`
- Fix `lib/dynamic/provider.tsx` same issues
- Fix `getSigner()` calls that don't exist on Dynamic v4 wallet type

## Files

- `src/components/dynamic/DynamicLoginButton.tsx`
- `src/lib/dynamic/provider.tsx`

## Acceptance Criteria

- [ ] No TypeScript errors in DynamicLoginButton
- [ ] No TypeScript errors in provider.tsx
- [ ] Login/logout flow works correctly
