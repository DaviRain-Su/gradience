# GRA-192: [Governance] Design dispute window + challenger slash for agent-arena

## Description

Define the on-chain governance rules for **Agent Arena disputes**, including a time-bounded dispute window, a challenger bond requirement, and a slashing mechanism for resolved disputes. This ensures judges remain accountable and agents can appeal unfair outcomes without opening the system to spam.

## Design Decisions

### 1. Dispute Window

- Opens immediately after `judgeAndPay` settles a task.
- Duration: **7 days** (`DISPUTE_WINDOW = 7 * 86400`).
- After the window closes, the settlement is final and non-reversible.

### 2. Challenger Bond

- Any address (losing agent, third-party watchdog, etc.) can open a dispute.
- Must post a bond equal to **10% of the task reward**.
- Bond is refunded if the dispute is upheld; slashed if the dispute is rejected.

### 3. Dispute Resolution Path

**Phase 1 — Rapid Council (≤ 48h)**

- A curated council of high-reputation judges reviews the evidence.
- Majority vote (> 50%) resolves simple cases.

**Phase 2 — Protocol Arbitration (escalation)**

- If council votes are split or the challenge is high-value (> 10 ETH equivalent), the case escalates to Protocol Arbitration.
- Protocol can override the original judge and reallocate rewards.

### 4. Slash & Reward Redistribution

- **If dispute upheld**:
    - Original judge is slashed **25% of their `minStake` equivalent**.
    - Winner is overturned; reward goes to the rightful agent (or refunded to poster if no valid submission).
    - Challenger bond is returned + a bounty of **5% of the slashed amount**.
- **If dispute rejected**:
    - Challenger bond is forfeited.
    - **50% of the forfeited bond** goes to the original judge as compensation.
    - **50%** goes to the protocol treasury.

## Implementation Plan

### Solidity Changes

1. **AgentArenaEVM**
    - Add `disputeTask(uint256 taskId, bytes32 reasonHash)` callable during the dispute window.
    - Add `resolveDispute(uint256 taskId, DisputeOutcome outcome)` callable by the dispute resolver.
    - Track `hasDispute[taskId]`, `disputeOpenedAt[taskId]`, `disputeBond[taskId]`.

2. **JudgeRegistry**
    - Add `slash(address judge, uint256 amount)` (owner/arena only).
    - Emit `JudgeSlashed(judge, amount, reason)`.

### Off-Chain Components

1. **Dispute Bot / Keeper**
    - Monitors `DisputeOpened` events.
    - Reminds council members of pending cases.
    - Automatically finalizes disputes after the resolution deadline.

2. **Indexer / Subgraph**
    - Index disputes by task, judge, and outcome for transparency dashboards.

## Acceptance Criteria

- [ ] Dispute window duration is configurable by protocol governance.
- [ ] Challenger must post a bond; bond logic is enforced in `AgentArenaEVM`.
- [ ] Original judge can be slashed and rewards redistributed upon upheld dispute.
- [ ] Council + escalation arbitration paths are documented and have corresponding access-control roles.
- [ ] Dispute events are indexed and queryable by the Chain Hub indexer.
