# GRA-178: Agent Detail page - connect Discover -> Profile flow

## Description
PRD core journey: Discover Agent -> click card -> see standardized Profile (identity/bio/links/reputation/capabilities) -> decide to delegate task.

Currently the DiscoverView shows a flat agent list with no click-through. The `/profile/[id]` route exists but:
1. Not linked from the app shell
2. Uses dark theme (bg-gray-900) inconsistent with main app (light #F3F3F8)
3. Shows mock profile data

## Fix Required
- Add Agent Detail panel/modal in DiscoverView (click agent card -> show detail)
- Include: name, bio, capabilities, reputation stats, wallet address
- Add "Invite to Chat" / "Delegate Task" action buttons
- Unify styling with main app theme
- Wire profile data to daemon API with demo fallback

## Files
- `src/app/app/page.tsx` (DiscoverView function)
- `src/app/profile/[id]/profile-content.tsx` (fix theme)
- New: Agent detail panel component
