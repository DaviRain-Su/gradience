# GRA-174: Integrate AutoLab as external Critic for KimiZ Agent

## Description

KimiZ is the Gradience-native LLM Agent framework powered by Kimi. To improve output reliability for tasks involving real-world web data, we integrate **AutoLabHQ** as an **external Critic**.

The Critic pattern:
1. KimiZ Agent generates a claim (e.g., "iPhone 15 is $799 on Amazon").
2. AutoLab Critic receives the claim + verification instructions.
3. AutoLab launches a headless browser, navigates to the target page, and extracts the ground-truth data.
4. The Critic returns a pass/fail verdict with structured evidence (screenshot, DOM snapshot, extracted values).
5. KimiZ Agent uses the Critic feedback to self-correct or finalize its answer.

This closes the loop between generative AI and verifiable browser automation, making KimiZ suitable for Agent Arena tasks that require web-grounded facts.

## Implementation Plan

### 1. Critic Interface Design
- Define `CriticRequest` / `CriticResponse` types.
- Support claim types: `price_check`, `availability_check`, `text_extraction`, `comparison`.

### 2. AutoLab Adapter
- Extend `apps/chain-hub/src/adapters/autolab-adapter.ts` (or create `kimiz-autolab-bridge.ts`).
- Map Critic requests to AutoLab skills (`web_scraping`, `form_filling`, `workflow_automation`).
- Handle API key injection via Key Vault.

### 3. KimiZ Integration
- Add `CriticTool` to KimiZ tool registry.
- Implement retry loop: generate → critic → reflect → regenerate (max 3 iterations).
- Cache successful critic runs to reduce API costs.

### 4. Agent Arena Alignment
- Ensure critic traces can be attached to `submitResult()` as `traceRef`.
- Make critic evidence downloadable by Judges during evaluation.

## Acceptance Criteria
- [ ] AutoLab Critic can verify a price claim against a live e-commerce page.
- [ ] Critic returns structured evidence (JSON + screenshot URL) within 30 seconds.
- [ ] KimiZ Agent uses Critic feedback to improve its answer in a multi-turn loop.
- [ ] Critic traces are persisted and referenced in task submissions.
- [ ] Unit tests cover claim parsing, AutoLab invocation, and retry logic.
