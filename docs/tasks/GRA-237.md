# GRA-237: gUSD 稳定币设计

## 背景

协议需要原生稳定币用于：

- 任务支付
- 收益分配
- 质押奖励

## 目标

设计并实施 gUSD 稳定币机制。

## 设计要点

### 1. 锚定机制

选项：

- 超额抵押（如 DAI）
- 算法稳定（如 FRAX）
- 法币储备（如 USDC）

### 2. 使用场景

- 任务 escrow
- Agent 质押
- 协议费用
- 治理代币配对

### 3. 合约设计

```rust
// 核心功能
- mint: 抵押铸造
- burn: 销毁赎回
- transfer: 标准 SPL
- freeze: 合规需求
```

## 阶段规划

### Phase 1: 设计 (2周)

- 机制选择
- 经济模型
- 安全审计准备

### Phase 2: 实现 (2周)

- 合约开发
- 测试网部署
- 审计

### Phase 3: 上线 (1周)

- 主网部署
- 流动性引导
- 集成到协议
