# GRA-229: Payment Service 接入真实 Evaluator

## 背景

当前 `payment-service.ts` 中的 `evaluateAndSettle()` 方法使用 mock 分数（85分）进行结算，而非真实的 LLM-as-Judge 评估结果。这导致：

- 任务结算不公平
- 无法反映实际工作质量
- 影响声誉计算准确性

## 目标

将 Payment Service 与 Evaluator 系统完全集成，使用真实的评估结果进行结算。

## 当前状态

```typescript
// apps/agent-daemon/src/services/payment-service.ts:485
const mockScore = 85; // Would come from Evaluator
// TODO: Integrate with real evaluator
```

## 实现方案

### 1. 依赖注入

- 在 PaymentService 构造函数中接收 Evaluator 实例
- 确保 Evaluator 已配置 LLM 客户端

### 2. 评估流程

```
收到提交 → 调用 Evaluator.evaluate(task, submission)
         → 获取各维度分数
         → 计算加权总分
         → 根据分数分配支付
```

### 3. 错误处理

- Evaluator 失败时：回退到人工评判队列
- 低置信度评估：标记为需要复核

## 验收标准

- [ ] Payment Service 使用真实 Evaluator 分数
- [ ] mockScore 被移除
- [ ] 集成测试通过
- [ ] 文档更新
