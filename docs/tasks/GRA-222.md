# GRA-222: Upgrade Web OWS Integration

## Description
The web frontend currently manages OWS state entirely in localStorage.
With the daemon now managing real OWS wallets (GRA-219), the frontend
should delegate wallet operations to the daemon and only handle UI +
token storage.

## Tasks
1. Update `useOWSBinding` hook:
   - On bind: call daemon `POST /api/v1/wallet/create` instead of
     local `OWSAgentWalletManager.bindMasterWallet()`
   - Store the returned wallet info (not raw keypair)
   - OWS DID now comes from real vault, not synthetic
2. Update `useOWSAgentRouter` hook:
   - `createSubWallet()` calls daemon API -> OWS SDK `createWallet()`
   - `signWithActiveSubWallet()` calls daemon `/api/v1/wallet/sign`
   - Sub-wallet list comes from daemon `GET /api/v1/wallet/list`
3. Update `OWSSdkClient`:
   - Keep local keypair path as offline fallback
   - Primary path: proxy all operations through daemon API
   - Passkey protection still used for storing OWS API key tokens
4. Add API key provisioning to My Agent tab:
   - After creating sub-wallet, auto-create API key with default policy
   - Show key once, store in Passkey-protected store
   - Show active keys list with revoke button
5. Remove `window.ows` provider detection (no browser extension exists)

## Acceptance Criteria
- [ ] Sub-wallet creation goes through daemon -> OWS SDK
- [ ] Signing uses daemon API with policy enforcement
- [ ] API key token stored in Passkey-protected store
- [ ] Offline fallback still works (local keypair)
- [ ] My Agent tab shows wallet + key management

## Dependencies
- GRA-219 (daemon OWS SDK integration)
- GRA-221 (API key management)
